HTML

  • HTML is used to create the structure of a webpage. It is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • HTML is the frame of the webpage (house analogy).
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML attribue- defines additional information about an element.
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users.
  • Child elements are elements nested inside other elements.
  • Parent elemtns are elements that hold all the other elements.
  • Semantic HTML elements- the name of the element describes its content and the role it plays on a webpage. They make your HTML easier to read by other developers and the browser.
  • Tag refers only to what appears inside the angled brackets.

CSS

  • CSS is a programming language used by developers to define how webpage content is presented to users.
  • Inline CSS- used when a developer wants to style an element directly in the HTML file.
  • Internal Style Sheet- contains CSS rules for the webpage in the head section of the HTML file.
  • External CSS Style Sheet- most common because it allows developers to keep all their CSS rules in a separate file, which makes changes easier.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside and element.

Git

  • git status: checks which branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.

JavaScript

  • JavaScript adds interactivity to our website.
  • JavaScript is the house once we can interact with it.
  • Browser Application Programming Interfaces (APIs)- built into web browsers, providing functionality such as dynamically creating HTML and setting CSS styles.
  • Third-Party APIs- allow developers to incorporate functionality in sites from other content providers, such as Twitter and Facebook.
  • Variables- containers that store values; use "let" keyword followed by the name you give to the variable
  • JavaScript is case sensitive!
  • Control Flow- the order in which a computer executes code. In JS it runs from top to bottom.
  • Conditional Statements- allow us to alter the control flow so that code executes only if a certain condition is met.
  • Array- a group of related data. A single variable that is used to hold a group of ideas or related data.
  • Index- unique number where the location of the data item in the array is identified.
  • for loops- use the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Function- a set of instructions that tells the computer how to perform a certain task. They are different from loops because they don't automatically execute when the JS file is run!
  • Requirements of a function: define the function, code block, call the function.
